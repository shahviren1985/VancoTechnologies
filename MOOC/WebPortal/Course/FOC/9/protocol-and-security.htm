HTTP

HTTP is based on the client/server principles. HTTP allows a client to establish connection with a server and make a request. The server accepts the connection initiated by the client and sends back the response.

HTTPS

HTTPS is secure version of HTTP protocol. The overall connection is secured using SSL certificate before actual communication happen.

FTP

This protocol allows users to transfer files from one computer to another computer. This method of file transfer is faster than that using HTTP.


Google chrome SSL option


  • Secure connection could be checked in address bar of browser.
  • Whenever we gave secure connection, it is prefixed with HTTPS protocol.
  • The address bar will also have one green colored lock sign.
  • Lock indicates that connection is secure. Green color indicates that the certificate used for secure connection is valid.
  • Invalid or expired certificates used to establish the connection is shown using red lock icons.
